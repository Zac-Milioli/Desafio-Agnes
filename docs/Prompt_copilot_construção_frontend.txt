Tenho o backend de um sistema de cadastro de projetos, clientes e atividades completo. O backend foi feito para ser um REST. Construa o frontend para a minha aplicação usando VueJS, este frontend irá se comunicar com o backend somente usando requests de API. O design das telas deve ser simples e moderno, usando o formato e a paleta da Microsoft. Não utilize nenhuma outra biblioteca de UI. O frontend não deve possuir nenhum sistema de login de usuários ou admin, e nenhuma autorização ou autenticação. Qualquer um que acessar será capaz de utilizar. O frontend deve conter 3 telas principais: uma tela de boas vindas, uma tela de projetos e uma tela de clientes e atividades. Caso tenha necessidade, pode criar telas a mais. As telas devem seguir as seguintes métricas:

- A tela de boas vindas deve conter a mensagem "Bem vindo!" escrita como título, podendo ter uma animação de fade-in ao abrir a tela. Deve haver um botão abaixo da mensagem de boas vindas com texto "Ir para "Meus Projetos"" que redireciona o usuário para a página de projetos.
- A tela de projetos, nomeada "Meus Projetos", é onde serão exibidos todos os projetos registrados. Ao abrir essa tela o frontend deve fazer a requisição para retornar todos os projetos disponíveis e colocar eles na tela. Os projetos devem estar separados em sessões diferentes por seu "status", que pode ser "Concluído", "Cancelado" e "Em andamento". Ao clicar em algum projeto, o usuário deverá ser encaminhado para a página "Clientes e Atividades de nome", substitua esse nome pelo "nome" do projeto selecionado. Deve haver na tela também um botão para excluír projetos, editar projetos e criar novos projetos, sendo que todas essas ações devem ser feitas através de requisições ao REST rodando no backend. Os atributos de projeto são "id" (que não precisa ser enviado ao criar um projeto), "nome" e "status" que pode ser "Concluído", "Cancelado" ou "Em andamento".
- A tela de clientes e atividades deve exibir, em diferentes blocos, os clientes conetados à esse projeto (são retornados pela API REST) e suas respectivas atividades (também retornado pela API REST) organizadas em cartões. Essa tela deve permitir criar, editar e excluir clientes (feito por requisições à API REST), sendo que os atributos do cliente são nome e projeto_id (é o id do projeto ao qual o cliente pertence). Também deve ser possível criar, editar e excluir atividades, sendo que os atributos de atividade são nome, projeto_id (id do projeto ao qual a atividade pertence) e id_cliente (id do cliente ao qual a atividade pertence)
